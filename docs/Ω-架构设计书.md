### 4.4 Pareto演化引擎

**理论**：多目标优化前沿维护，驱动系统向更低自由能方向演化。七个维度的非支配解空间持续更新。

**工程实现**：

```zig
const ParetoFrontier = struct {
    points: []Point,          // 非支配解前沿点集
    dimensions: []Dimension,   // 目标维度（初始7维）
    capacity: usize,           // 前沿容量（自适应）
};
```

**支配判断**：p dominates q ⇔ ∀d: p[d]≥q[d] ∧ ∃d: p[d]>q[d]

**插入算法**：
```
insert(candidate):
  for each point in frontier:
    if candidate dominates point → remove point
    else if point dominates candidate → reject
  add candidate to frontier
  if |frontier| > capacity → prune (保留多样性最大子集)
```

**饱和检测**（三判据）：

| 判据 | 方法 | 条件 |
|------|------|------|
| 时间判据 | 连续T步无新支配点 | T=1000 |
| 随机扰动 | 前沿点±1%扰动后无法产生新支配点 | — |
| 多起点 | 随机初始化重新演化，前沿差异<ε | — |

**维度权重**：w_i = softmax(avg_improvement_i / temperature)，全内生，无人工设定

**状态**：✅ pareto_front.zig + saturation_detector.zig + dimension_weights.zig 完整实现

### 4.5 层级跃迁引擎

**理论**：当微观演化在当前维度空间饱和，通过打包→映射→展开三步骤跃迁到新维度空间继续降低自由能。

**工程实现**：

```
饱和触发 → 打包(Packing): 当前层稳定结构抽象为高阶对象
    → 映射(Mapping): 新维度态射关系建立
    → 展开(Unfolding): 新层Pareto演化重启
    → 验证: F(新层) < F(旧层)? 否则回退
```

**回退条件**：
1. 新层演化T步后F(新层) >= F(旧层)
2. 新层出现旧层不存在的结构矛盾
3. 新层Pareto前沿发散

**状态**：✅ macro_bootstrap.zig + layer_transition.zig 完整实现---

## 十一、设计原则（白皮书第11章精要）

1. **Δ唯一定理**:所有运算归约到Δ(x,y)=f(x)-g(y)
2. **自生生长**:不引入新计算范式
3. **格码同构**:每个构造对应CDL格结构
4. **双态同显**:路径态与规则态是同一实体的两种状态
5. **自由能驱动**:F=αF_fit+βF_comp+γF_cons作为唯一优化目标
6. **全分布式**:无全局周期/无中心调度器/无阻塞
